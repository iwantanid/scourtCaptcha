# 대법원 나의 사건 검색 Captcha Decoder - 기술 아키텍처 설명서

이 설명서는 컴퓨터공학 및 인공지능을 전공하는 학부생과 딥러닝 입문자를 위해 작성했습니다. 이 프로젝트에 사용한 CaptchaCNN 모델의 구조와 레이어별 특징 추출 메커니즘, 그리고 최적화 학습 파이프라인의 원리를 차근차근 설명합니다.

---

## 데이터 파이프라인 (Data Pipeline)

모델이 받아들이는 캡차 이미지는 단순한 그림 파일이 아닙니다. 딥러닝 연산에 최적화된 4차원 수학적 텐서(Tensor) 데이터로 변환되어 모델에 전달됩니다.

```
[입력 이미지] (121 x 46 RGB)
       │
       ▼ (1. Grayscale 변환)
[1채널 흑백 이미지] (121 x 46 x 1)
       │
       ▼ (2. Resize 보간법)
[고정 크기 이미지] (120 x 40 x 1)
       │
       ▼ (3. 정규화 Min-Max scaling)
[정규화 픽셀 값] (0.0 ~ 1.0 실수형 행렬)
       │
       ▼ (4. 배치 차원 확장 및 PyTorch 텐서 변환)
[최종 모델 입력 텐서] (B, 1, 40, 120)  -- B: 배치 크기
```

*   **Grayscale 변환**: 색상 정보는 글자 형태를 판독하는 데 크게 중요하지 않습니다. 따라서 채널을 1개로 줄여 메모리와 연산 효율을 약 3배 높였습니다.
*   **실시간 데이터 증강 (On-the-fly Data Augmentation)**:
    *   **RandomAffine 기법**을 활용해 학습용 데이터에만 실시간으로 미세 회전(±5도), 미세 평행 이동(±5%), 미세 배율 조정(0.95~1.05)을 무작위로 적용합니다.
    *   이 과정을 거치면 캡차 이미지에 인위적인 줄(Strike line)이 그어지거나 글씨 왜곡이 심한 실제 상황에서도, 과적합(Overfitting) 없이 글자를 판독하는 강력한 일반화 성능(Generalization)을 기르게 됩니다.

## 모델 아키텍처 (CaptchaCNN Architecture)

CaptchaCNN 모델은 합성곱 블록(Convolutional Block) 4개, 완전 연결 병목 레이어(Dense Bottleneck) 1개, 그리고 병렬 분류 헤드(Parallel Classification Heads) 6개로 연결한 독립 분기 구조입니다.

```
                    [입력: 1 x 40 x 120]
                             │
                  ┌──────────┴──────────┐
                  │  Conv Block 1 (32)  │  --> MaxPool (20 x 60)
                  └──────────┬──────────┘
                             │
                  ┌──────────┴──────────┐
                  │  Conv Block 2 (64)  │  --> MaxPool (10 x 30)
                  └──────────┬──────────┘
                             │
                  ┌──────────┴──────────┐
                  │  Conv Block 3 (128) │  --> MaxPool (5 x 15)
                  └──────────┬──────────┘
                             │
                  ┌──────────┴──────────┐
                  │  Conv Block 4 (256) │  --> MaxPool (2 x 7)
                  └──────────┬──────────┘
                             │
                  ┌──────────┴──────────┐
                  │  Flatten & Dense    │  --> 512-dim 병목 벡터
                  └──────────┬──────────┘
                             │
        ┌────────┬──────────┼──────────┬────────┬────────┐
        ▼        ▼          ▼          ▼        ▼        ▼
     [Head 1] [Head 2]   [Head 3]   [Head 4] [Head 5] [Head 6]
      (10)     (10)       (10)       (10)     (10)     (10)
```

### 합성곱 블록 (Convolutional Block)

각 합성곱 블록은 다섯 가지 핵심 레이어로 구성되어 이미지 특징을 다방면으로 추출합니다.

*   **합성곱 레이어 (nn.Conv2d)**
    *   `3x3` 크기의 필터 커널(Kernel)이 이미지 위를 쓸어가며 공간적 특징을 뽑아냅니다.
    *   채널 수를 순차적으로 확장(32에서 64, 128, 256)해 나갑니다. 앞쪽 블록은 경계선이나 테두리 같은 단순한 시각 정보(Low-level feature)를 포착하고, 뒤쪽 블록은 구체적인 글자 형상에 가까운 고수준 추상 정보(High-level feature)를 포착하도록 설계했습니다.
*   **배치 정규화 (nn.BatchNorm2d)**
    *   미니배치(Mini-batch) 단위로 중간 레이어의 출력값을 평균 0, 분산 1로 표준화하여, 학습이 진행될 때 데이터 분포가 흔들리는 **내부 공변량 변화(Internal Covariate Shift)** 문제를 예방합니다.
    *   배치 정규화를 거치면 가중치 초깃값에 지나치게 민감하게 반응하지 않으며, 경사 하강법 기반의 가중치 학습 속도가 한층 안정되고 빨라집니다.
*   **ReLU 활성화 함수 (nn.ReLU)**
    *   비선형 활성화 함수 $f(x) = \max(0, x)$를 적용해 음수 값을 0으로 깎아냅니다. 신경망에 비선형성을 제공하여 모델이 단순한 일차원 경계를 넘어 복잡한 글자 경계선까지 구분할 수 있게 돕습니다.
*   **최대 풀링 (nn.MaxPool2d)**
    *   `2x2` 영역에서 가장 큰 값만 남기는 다운샘플링(Spatial Downsampling)을 진행합니다.
    *   이미지의 공간 해상도를 절반으로 줄여 불필요한 계산량을 낮추고, 글자 위치가 미세하게 비틀어져도 안정적으로 판독해내는 **공간적 평행이동 불변성(Translation Invariance)**을 만들어냅니다.
*   **드롭아웃 (nn.Dropout)**
    *   학습할 때 일부 뉴런을 무작위로 비활성화(10%~25%)하여, 특정 노드 집단끼리만 뭉쳐서 과하게 동조하는 현상(Co-adaptation)을 인위적으로 차단합니다. 앙상블 모델을 학습시키는 듯한 규제(Regularization) 작용을 하여 과적합을 방지합니다.

### 512차원 병목 임베딩 (Dense Bottleneck)

최종 합성곱 블록을 빠져나온 출력값은 `[256, 2, 7]` 크기의 텐서입니다. 이를 1차원으로 곱게 일렬 정렬하여 **3,584차원 특징 벡터**로 펼칩니다(Flatten). 그 뒤 완전 연결(Fully Connected) 레이어를 통과시켜 **512차원의 핵심 특징 벡터(Bottleneck Embedding)**로 부피를 압축합니다. 이 512차원 벡터 공간에는 6자리 숫자의 기하학적 형태 정보가 요약되어 보존됩니다.

### 병렬 다중 헤드 출력 구조 (Multi-Head Output Topology)

이 모델의 핵심 설계입니다. 순서 정보를 학습하기 위해 주로 쓰는 복잡한 RNN이나 어텐션(Attention) 레이어 대신, **6개의 독립된 선형 분류 노드 헤드(`head1`부터 `head6`, 각각 `nn.Linear(512, 10)`)**를 병렬로 나란히 엮었습니다.

*   **특징과 장점**
    1.  모든 분류 헤드가 512차원 병목 임베딩을 공유합니다. 각 헤드는 자신이 담당한 자리의 숫자(0~9 클래스)를 판독하도록 별개로 훈련되며, 각 위치에서 발생한 손실을 따로 역전파합니다.
    2.  앞 자릿수부터 순서대로 계산해야만 하는 RNN 계열 모델과 비교했을 때, 병렬 헤드 구조는 모든 자릿수의 계산을 **GPU 텐서 병렬 연산으로 일시에 처리**합니다. 덕분에 연산 지연 속도가 1ms 수준으로 매우 빠릅니다.

## 최적화 및 훈련 파이프라인 (Optimization & Training)

### 손실 함수 (Loss Function)

자릿수마다 분류 확률 예측치와 정답지 사이의 오차를 구하기 위해 **교차 엔트로피 손실 함수(Cross-Entropy Loss)**를 썼습니다. 전체 모델의 총 손실값은 아래와 같이 6개 분류 헤드에서 계산된 개별 손실의 총합(Summation)으로 규정합니다.

$$\text{Total Loss} = \sum_{i=1}^{6} \text{Loss}(\text{head}_i, \text{target}_i)$$

오차 역전파(Backpropagation)가 일어날 때, 이 합산 손실값의 기울기(Gradient)가 연산 그래프를 거슬러 모델 전체로 고루 흘러 들어갑니다. 이 과정을 통해 합성곱 레이어와 완전 연결 레이어의 가중치 갱신이 동시에 일어납니다.

### 가중치 최적화와 스케줄러 (Optimizer & LR Scheduler)

*   **AdamW 옵티마이저**: 학습 방향을 관리하는 모멘텀(Momentum) 기법과 가중치 크기에 따라 학습 폭을 조절하는 적응형 학습률(Adaptive Learning Rate) 기법을 엮고, 가중치 값의 과도한 팽창을 누르는 L2 규제(Weight Decay)를 얹은 AdamW 옵티마이저를 씁니다.
*   **ReduceLROnPlateau 스케줄러**: 검증 데이터 손실값이 5 에포크 동안 나아지지 않고 정체되면 학습률을 0.5배씩 낮춥니다. 학습의 끝자락에서 모델이 최적 지점을 지나쳐 헤매지 않고 정답 근처로 촘촘히 진입하도록 보조하는 장치입니다.

## 데이터 중심 AI 관점의 모델 튜닝 성과 (Data-Centric AI Proof)

이 프로젝트는 모델 구조를 계속 복잡하게 개정하는 방식보다, **학습용 데이터 자체의 노이즈와 품질 오류를 바로잡는 일**이 모델 최종 성능 상승에 얼마나 큰 역할을 하는지 잘 증명해 줍니다.

1.  **라벨 노이즈(Noisy Labels) 추적**
    *   초기 1차 학습 시기에 모델은 검증 데이터 6자리 일치율 98.19% 수준에서 한계를 마주하며 더 나아지지 못했습니다.
    *   판독에 오답을 낸 11건의 이미지를 직접 열어 세밀하게 눈으로 확인했습니다. 그 결과 **모델이 내놓은 판독 결과가 실제 시각 정보와 완전히 일치**했으며, 정답지(`answer_key.txt`)의 정답 텍스트에 오타가 끼어 있던 라벨 노이즈(Label Noise)였음을 찾아냈습니다. (예: 실제 이미지 글자 '811699'가 정답지에 '211699'로 잘못 받아적혀 있음)
2.  **데이터 정제에 따른 성과**
    *   오타가 확인된 11개 이미지의 오기입 정답을 올바른 텍스트로 고쳐 정제한 뒤 재학습을 시켰습니다. 그러자 **단 13 에포크(약 1분 15초)** 만에 6자리 전체 일치 정확도 **99.34%** (글자 단위 **99.89%**)를 달성하며 목표 수준을 넘어 학습이 조기 종료되었습니다.

> 이 시스템은 적절하게 연계된 4단 CNN 특징 추출 엔진, 다중 병렬 분류 구조, 그리고 데이터 중심(Data-Centric)의 정교한 품질 관리가 조화를 이뤘을 때, 적은 수준의 파라미터(약 200만 개)만으로도 상용급 완성도의 정확도를 신속하게 뽑아낼 수 있음을 보여주는 모범적 실증 사례입니다.
